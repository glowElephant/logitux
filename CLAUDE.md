# logitux

리눅스에서 로지텍 마우스 버튼을 **마우스 도식을 보고 클릭만으로** 매핑하는 사내 배포용 도구.
Logi Options+의 리눅스 대안 — 단, 핵심 차별점은 **시각적 버튼 매핑 UX**와 **비기술자도 쓰는 쉬운 배포**.

## 목표 (한 줄)

사내 리눅스 사용자 누구나, Solaar 같은 복잡한 도구 없이, 마우스 그림에서 버튼을 클릭해 키조합을 등록할 수 있게 한다.

## 아키텍처 (의도된 설계)

- **GUI (PySide6/Qt)** — `QGraphicsView`로 마우스 SVG 도식 위에 버튼 핫스팟·연결선·라벨을 얹음. 버튼 클릭 → 키조합 입력 → 저장.
- **백엔드 (Solaar/HID++)** — 실제 버튼→키조합 매핑 엔진은 Solaar를 **뒤에 숨겨** 활용. 사용자에게 Solaar 룰 편집을 절대 노출하지 않는다.
- **도식 데이터** — 마우스 모델별 SVG + 버튼 좌표(핫스팟) 메타데이터. MX Master 3/4부터, 모델은 데이터 추가만으로 확장.
- **상시 실행** — MX 시리즈는 온보드 저장이 없어 매핑 변환 소프트웨어가 항상 떠 있어야 함 (autostart 등록).

## 에이전트 가드레일 (피할 것)

- **Solaar 룰 직접 편집을 UI에 노출 금지** — 그건 백엔드 구현 디테일. 사용자는 그림과 클릭만 본다.
- **MX 마우스 온보드 저장 가정 금지** — MX Master는 온보드 메모리가 없다. 매핑은 소프트웨어 상시 실행 전제. (온보드 저장은 로지텍 G 게이밍 시리즈 전용)
- **마우스 모델 하드코딩 금지** — 모델은 SVG+좌표 데이터로 외부화. 새 모델 추가 시 코드 수정 없이 데이터만.
- **절대 경로 하드코딩 금지** — `__file__`, 환경변수, 설정 파일 사용.
- **X11 전용 가정 금지** — Wayland 사용자도 대상. 디스플레이 서버 의존 코드는 분기.
